# moomoo 植入内容工作台

把《moomoo 植入内容策略》这套 deck 变成一个在线工作台：BD 自行勾选需要的页面、导出 PDF 分享给 KOL；你可以在网页里直接编辑每页内容、新增功能页。

## 目录结构
- `index.html` — 工作台本体（自带样式和逻辑，无构建、无依赖）
- `content.js` — **所有页面内容的源文件**（内嵌加载，唯一真相来源）
- `assets/` — 手机截图、封面实景图等图片素材

## 打开方式
内容已内嵌进 `content.js`，所以：
- **本地**：直接双击 `index.html` 即可打开使用（无需服务器）
- **线上**：部署后访问 `/kol-workbench/`

## 访问密码与角色（两种身份）
首次进入需输入密码，**输入的密码决定身份**：
- **编辑者密码**（只给你自己）：默认 `moomoo-edit-2026` —— 可编辑文字、新增功能页、导出内容
- **查看者密码**（发给同事）：默认 `moomoo2026` —— 只能勾选页面 + 导出 PDF，看不到也用不了任何编辑功能

改密码：编辑 `index.html` 顶部这两行：
```js
const EDITOR_PASSWORD = 'moomoo-edit-2026';
const VIEWER_PASSWORD = 'moomoo2026';
```
右上角「退出」可清除当前身份、重新输密码切换。

> 关于安全：这是「软门禁」。真正的「编辑权限」是**谁能把 `content.js` 提交到仓库**——只有你有 GitHub 权限，同事即使绕过门禁在本地改也只影响他自己那份、传不到别人。门禁的作用是把编辑 UI 对同事隐藏、防误操作。若要**真正的访问控制**（限定邮箱才能打开），见下方「进阶」。

## 三件事怎么做

### 1. BD 勾选页面 → 导出 PDF
- 左侧勾选要包含的页面，拖动 `⋮⋮` 调整顺序，双击某行跳转到对应页
- 未勾选的页在画布里会变灰并标注「未选中 · 不会导出」
- 点右上 **🖨️ 导出 PDF** → 在系统打印对话框选「另存为 PDF」。版式 16:9 横版，每页一张幻灯片
- 勾选状态保存在各自浏览器本地，互不影响

### 2. 编辑每页内容
- 点右上 **✏️ 编辑模式**，页面文字直接点击修改
- 卖点 / 标签 / 场景卡片末尾有 `＋` 可新增，每项悬停出现 `×` 可删除
- 功能页手机区域：编辑模式下点击即可上传替换截图
- 「点击观看」按钮：编辑模式下填入视频链接，即变成可点链接
- 改完点 **🚀 保存并发布** → 一键提交到仓库，约 1 分钟后同事刷新即可看到（见下「一键发布」）
- 或点 **⬇️ 导出内容** → 下载 `content.js` 手动覆盖提交（备份/无 Token 时用）

> 编辑先暂存在你本地浏览器，点「保存并发布」或导出提交后才对所有人生效。点 **↺ 重置** 可清除本地修改、恢复到仓库已发布版本。

## 一键发布（推荐，免去导出/git）
编辑者点 **🚀 保存并发布**，工作台会通过 GitHub API 直接把 `content.js` 提交到仓库——不用导出文件、不用碰 git。首次会让你粘贴一个 GitHub Token（只存在你这台电脑的浏览器里）。

**创建 Token（一次性，约 2 分钟）：**
1. 打开 <https://github.com/settings/tokens?type=beta>（Fine-grained tokens）→ **Generate new token**
2. **Repository access** → Only select repositories → 选 `PurpleOPQ/market-monitor`
3. **Permissions** → Repository permissions → **Contents** 设为 **Read and write**
4. 生成后复制 token（`github_pat_...`），第一次点「保存并发布」时粘贴即可

- Token 只保存在你本机浏览器（localStorage），点右上「退出」会一并清除。
- 想作废随时去同一页面 Revoke；建议设较短有效期。
- 同事是查看者，看不到「保存并发布」按钮，也不需要 Token。

### 3. 新增功能介绍页
- 点左下 **＋ 新增功能页**，追加一张空白功能页并自动进入编辑模式，填好内容、传截图
- 同样记得 **导出内容** 并提交，才会成为正式版本

## 部署 + 分享给同事（推荐 Cloudflare Pages）
本仓库已在用 Cloudflare Pages（见根目录 `DEPLOY.md`）。把改动 push 到 GitHub 后会自动部署，工作台地址即：

```
https://<你的-pages-域名>/kol-workbench/
```

把这个链接 + 密码（默认 `moomoo2026`）发给同事即可，他们打开 → 输密码 → 直接用。

### 进阶：真正的访问控制（可选）
若想要「只有指定同事能进」的硬权限，用 **Cloudflare Access**（免费）：
Cloudflare 控制台 → Zero Trust → Access → Applications → 添加一个 Self-hosted 应用，
路径设为 `/kol-workbench/*`，策略选「Emails」并填同事邮箱。
之后访问该路径会要求邮箱验证码登录，比前端密码安全得多。

## 注意
- 上传替换的图片会以 base64 内嵌进 `content.js`，会让文件变大。长期使用建议把图片放进 `assets/`，再在 `content.js` 里把对应 `image` 字段改成相对路径（如 `assets/phone-xxx.png`），体积更小、更好维护。
